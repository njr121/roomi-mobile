# Phase 20 — 예약중복방지 (동시 예약 생성 시 중복 발생 방지)

> 완료된 작업. 참고용 기록.
> roomi-api 도메인(백엔드 로직).

---

## 방향

같은 객실·같은 날짜로 거의 동시에 예약 요청이 들어오면, 중복 확인(conflict check)과 생성(create) 사이의 시간차(TOCTOU)로 양쪽 다 통과해서 중복 예약이 생기는 문제를 막는다.

---

## 구성요소 상세

| 항목 | 내용 |
|---|---|
| 원인 | 중복 확인 쿼리와 생성 쿼리가 별개 트랜잭션 — 동시 요청 시 둘 다 "중복 없음"으로 통과 가능 |
| 해결 | `prisma.$transaction(..., { isolationLevel: Serializable })`로 확인+생성을 하나의 트랜잭션으로 묶음 |
| 충돌 처리 | 직접 던진 `"BOOKING_CONFLICT"`와 Prisma 직렬화 실패 코드(`P2034`) 둘 다 409로 응답 |

## 변경 파일

`app/api/bookings/route.ts`

## 완료 기준

동시 curl 요청 테스트 — 하나는 200, 하나는 409 확인 완료.
